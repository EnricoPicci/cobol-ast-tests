# COBOL AST Parser Examples

Educational examples of parsing COBOL source code using AST (Abstract Syntax Tree) parsers. The project demonstrates AST parsing techniques and follows an AI-assisted prompt-driven development workflow.

## Project Goal

Demonstrate how AST parsers can analyze and transform COBOL source code. The project takes a multi-language approach: **Python** is implemented first, with plans to add **TypeScript/Node.js**, **Go**, and **Java**. All languages share the same COBOL sample files to allow comparison across ecosystems.

### COBOL Migration Context

AST-based analysis can help detect and fix issues that arise when migrating COBOL applications across platforms — for instance, the endianness problems that occur when moving from AIX (big-endian) to Linux x86 (little-endian). ## How This Project Is Built — Prompt-Driven Development

Every artifact in this project — code, documentation, even the prompts themselves — was generated by giving prompts to [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

The [`prompts/`](prompts/) directory contains numbered files that record the exact sequence of instructions used to build the project from scratch. Following them in order shows how the project evolved step by step.

Many prompts are *meta-prompts* — prompts that generate other prompts. For example:

1. `04-create-prompt-to-choose-python-ast-parser.md` — a meta-prompt that refines a prompt for choosing a Python AST parser
2. `05-choose-python-ast-parser.md` — the resulting prompt, which evaluates parsing libraries and recommends the best fit

This two-step pattern (meta-prompt followed by execution prompt) repeats throughout the sequence. See the [`prompts/`](prompts/) directory for the full history.

## COBOL Samples

The [`samples/`](samples/) directory contains shared COBOL source files (`.cob`) used by all language implementations. Samples are organized by topic — for example, [`samples/endianness/`](samples/endianness/) contains programs that demonstrate endianness problems (with and without issues) when migrating COBOL from big-endian AIX to little-endian Linux. All languages read from this single shared location; sample files are never duplicated into language-specific directories.
